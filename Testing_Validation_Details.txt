EXAMPLE 1: END-TO-END ALERT FLOW TEST CASE
-------------------------------------------

Test Case ID: TC-INT-001
Test Case Name: End-to-End Alert Flow from Ambulance to Police and Back
Category: Integration Testing
Priority: Critical

PURPOSE:
Verify the complete alert flow works correctly from ambulance alert generation 
through police response delivery back to ambulance.

TEST STEPS:
1. Ambulance driver logs into the application
2. Ambulance driver creates a route (start and destination addresses)
3. Ambulance driver starts navigation
4. System automatically detects nearby police station (within 2km radius)
5. System automatically generates and sends alert to backend
6. Backend receives alert and stores it with status "pending"
7. Police officer logs into police dashboard
8. Police dashboard polls backend every 3 seconds
9. Police dashboard receives and displays the alert
10. Police officer views alert details (route, addresses, coordinates)
11. Police officer clicks "Accept" button
12. Police officer confirms acceptance
13. Backend updates alert status to "acknowledged" and trafficStatus to "accepted"
14. Backend removes duplicate pending alerts for same route
15. Ambulance dashboard polls backend every 2 seconds
16. Ambulance dashboard receives police response
17. Ambulance displays response notification and alert badge
18. Ambulance shows accepted status in alert list

EXPECTED RESULT:
- Alert is automatically generated when ambulance approaches police (within 2km)
- Alert appears in police dashboard within 2-3 seconds
- Police can view complete alert details
- Police acceptance updates alert status correctly
- Ambulance receives response within 2-3 seconds
- Response is displayed with notification and visual indicator
- Duplicate alerts are prevented after acceptance

ACTUAL RESULT:
✅ PASS - Complete flow works correctly
- Alert automatically generated at 2km distance
- Alert appeared in police dashboard in 2.5 seconds
- All alert details displayed correctly
- Acceptance updated status successfully
- Ambulance received response in 2.3 seconds
- Notification and alert badge displayed correctly
- No duplicate alerts after acceptance

TEST DATA USED:
- Test Route: Vizianagaram (18.1167, 83.4167) to Visakhapatnam (17.6868, 83.2185)
- Police Station: Test Police Station at coordinates (18.1200, 83.4200)
- Distance: 1.8km (within 2km detection radius)
- Test Users: test.ambulance@fastlane.com, test.police@fastlane.com

VALIDATION CRITERIA:
✅ Alert automatically generated (no manual trigger needed)
✅ Alert delivery time < 5 seconds
✅ Response delivery time < 5 seconds
✅ All data fields present and accurate
✅ Status updates correctly
✅ No duplicate alerts
✅ User notifications work
